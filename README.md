# 🐍 Python (Py4E): A DevOps Evolution

I am documenting my progress through a **DevOps and Automation** lens transforming basic syntax into powerful infrastructure tools.

---

## 🧭 Learning Path & Milestones

This project follows the core modules of the Py4E curriculum, with dedicated scripts for each milestone:

### 1. Variables, Expressions, and Flow
* **Concepts:** Conditional execution (`if/else`), `try/except` for robust code, and loops (`for/while`).
* **DevOps Perspective:** Writing logic for automated decision-making (e.g., "If disk space > 90%, trigger cleanup").

### 2. Python Data Structures
* **Concepts:** Strings, Files, Lists, Dictionaries, and Tuples.
* **DevOps Perspective:** Parsing configuration files, managing lists of server IPs, and using dictionaries to store structured system metadata.

### 3. Using Python to Access Web Data
* **Concepts:** Regular Expressions (Regex), Networked Programs (Sockets/HTTP), and Web Services (JSON/XML).
* **DevOps Perspective:** Scraping logs with Regex, interacting with REST APIs, and automating health checks for microservices.

### 4. Using Databases with Python
* **Concepts:** SQL, SQLite, and Data Modeling.
* **DevOps Perspective:** Building local databases to track asset inventories, audit logs, or system performance history.

---

## 🛠️ The DevOps Toolbox
Throughout this course, I am maintaining high engineering standards:
* **Clean Code:** Adhering to PEP 8 standards for readability.
* **Version Control:** Using Git for every minor logic change to track the evolution of scripts.
* **IDE:** Developing in **Visual Studio Code** with Pylance for professional-grade linting.
